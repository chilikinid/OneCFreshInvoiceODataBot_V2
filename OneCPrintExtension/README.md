# OneCPrintExtension

Заготовка расширения 1С для получения штатных печатных форм в PDF по HTTP-запросу от бота.

Это не собранный `.cfe`, а исходники для создания расширения в Конфигураторе/EDT.

## Структура

- `src/CommonModules/ODataBotПечатьСервер.bsl` - общий модуль сервера.
- `src/HTTPServices/odata-bot-print/HTTPСервис_Модуль.bsl` - модуль HTTP-сервиса.
- `examples/request-example.json` - пример запроса от бота.
- `examples/response-example.json` - пример JSON-ответа, если PDF возвращается в base64.

## HTTP-контракт

Бот отправляет `POST`:

```json
{
  "entitySet": "Document_СчетНаОплатуПокупателю",
  "refKey": "11111111-1111-1111-1111-111111111111",
  "printForm": "Счет",
  "format": "pdf"
}
```

Расширение должно вернуть:

- `Content-Type: application/pdf` и байты PDF;
- или JSON с `contentBase64`.

## Как создать расширение

1. Открыть базу в Конфигураторе.
2. Создать расширение, например `ODataBotPrint`.
3. Добавить общий модуль `ODataBotПечатьСервер`.
   - Сервер: Да.
   - Вызов сервера: Да.
   - Экспорт: Да.
4. Вставить код из `src/CommonModules/ODataBotПечатьСервер.bsl`.
5. Добавить HTTP-сервис `odata-bot-print`.
6. Добавить шаблон `/print` с методом `POST` и обработчиком `Печать`.
7. Вставить код из `src/HTTPServices/odata-bot-print/HTTPСервис_Модуль.bsl`.
8. Опубликовать HTTP-сервис.

## Что нужно адаптировать

В `ODataBotПечатьСервер.bsl` нужно заполнить функции:

- `СформироватьСчетНаОплату`
- `СформироватьРеализациюИлиУПД`
- `СформироватьСчетФактуруВыданный`

Именно эти функции должны вызвать штатные печатные формы вашей конфигурации 1С и вернуть `ТабличныйДокумент`.

Остальная часть уже готова: прием HTTP-запроса, поиск документа по GUID, сохранение табличного документа в PDF и возврат PDF.

## Настройки бота

```json
{
  "Processing": {
    "GeneratePdf": true
  },
  "PrintService": {
    "Enabled": true,
    "Url": "https://.../hs/odata-bot-print/print",
    "Login": "",
    "Password": "",
    "TimeoutSeconds": 120
  }
}
```
