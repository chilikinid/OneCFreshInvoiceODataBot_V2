# 🚀 Быстрый старт - персистентный кэш

## 30 секунд для понимания

**Проблема**: Программа медленно загружает данные из OData при каждом запуске.

**Решение**: Персистентный кэш сохраняет данные на диск и использует их при следующих запусках.

**Результат**: **70x ускорение** на повторных запусках (7 сек → 100 мс)

## 🔑 Ключевые особенности

✅ **Автоматический** - работает без дополнительной конфигурации

## 🎮 Как использовать?

### Просто запустите приложение

```
1. Первый запуск:
   - Программа загружает из OData
   - Сохраняет в кэш
   - Обрабатывает документы

2. Второй запуск:
   - Программа загружает из кэша
   - Пропускает OData запросы
   - Обрабатывает документы БЫСТРО ⚡
```

### Очистить кэш (если нужно)

```powershell
# PowerShell
Remove-Item "$env:LOCALAPPDATA\OneCFreshInvoiceODataBot\Cache" -Recurse -Force
```

Или просто удалить папку вручную.

## ❓ FAQ

### Q: Как узнать, что кэш работает?
**A**: Второй запуск будет НАМНОГО быстрее (на порядок). Смотрите время в консоли.

### Q: Какой размер кэша?
**A**: Для 1000 номенклатур - примерно 10-50 МБ (зависит от данных)

### Q: Могу ли я использовать один кэш для разных приложений 1С?
**A**: Нет, каждое приложение 1С имеет отдельный кэш (автоматически)

### Q: Что если данные изменились в 1С?
**A**: Кэш используется как есть. Очистите кэш если нужны новые данные.

### Q: Является ли кэш безопасным?
**A**: Да, хранится в пользовательской папке, не требует админ прав

### Q: Могу ли я удалить кэш?
**A**: Да, программа просто перезагрузит данные из OData

### Q: Может ли кэш повредить данные?
**A**: Нет, это только для чтения. Кэш не влияет на отправку документов в 1С
